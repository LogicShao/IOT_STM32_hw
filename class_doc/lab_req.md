# 直流电能表开发板实验 — 要求与目标总结

## 一、实验概述

本实验基于 **STM32F070CBT6TR + HT7017(F 版)** 直流电能表开发板，要求学生从零搭建嵌入式开发环境，完成三个核心功能模块的设计与实现：**红绿灯控制、模拟电梯、直流电表数据采集**。

---

## 二、实验目的

1. 掌握 **STM32CubeMX + CLion** 嵌入式开发流程，熟悉 STM32F070CBT6TR 开发板使用。
2. 理解并掌握 **GPIO、定时器、外部中断** 等外设配置，掌握**有限状态机 (FSM)** 在嵌入式系统中的应用。
3. 学习 **GN1650 数码管驱动芯片** 与 **HT7017 直流计量芯片** 工作原理，掌握 **UART、I2C** 通信协议。
4. 了解 **RS485 半双工通信**及 **HC-05 蓝牙模块**，实现开发板与上位机/手机的数据交换。
5. 培养综合调试能力（硬件连接、代码烧录、串口调试、系统联调）。

---

## 三、硬件平台

### 3.1 核心组件

| 组件 | 说明 |
|---|---|
| **主控 MCU** | STM32F070CBT6TR，48MHz，64KB Flash，8KB RAM |
| **计量芯片** | HT7017，通过 USART1（4800bps, 9bit, Odd 校验）与 MCU 通信 |
| **显示驱动** | GN1650 数码管驱动芯片，3 组数码管（编号 1/2/3） |
| **存储** | M24256E EEPROM (32KB)，I2C 接口 |
| **通信** | RS485 (USART4, 38400bps)，HC-05 蓝牙模块 |

### 3.2 外设资源

| 外设 | 功能 | 引脚/配置 |
|---|---|---|
| **LED** | 红/绿/黄 + 测试 LED | PB1(RED), PB0(GRE), PA7(YEL), PA11(TEST) |
| **蜂鸣器** | 无源蜂鸣器，PWM 驱动 | PA2 (TIM15_CH1), 4kHz |
| **按键** | 4 个机械按键 (UP/DOWN/LEFT/RIGHT) | PB4, PA15, PB3, PB5 (EXTI 中断) |
| **数码管** | 3 组，模拟 I2C 协议驱动 | 开漏输出 |
| **I2C** | EEPROM 通信 | PB6(SCL), PB7(SDA), 100kHz |
| **USART1** | HT7017 计量芯片 | PA9(TX), PA10(RX), 4800-9-Odd |
| **USART4** | RS485 通信 | PA0(TX), PA1(RX), 38400-8-None |
| **TIM6** | 周期中断（心跳/倒计时） | 2Hz (500ms 周期) |
| **TIM15** | PWM 蜂鸣器驱动 | 4kHz |

## 四、三大功能模块

### 4.1 红绿灯控制系统

**核心设计模式**：有限状态机 (FSM)，分离为「运行模式」与「设置模式」。

**功能需求**：

| 需求 | 描述 |
|---|---|
| **自动倒计时切换** | 红灯→绿灯自动循环，数码管 1 显示当前倒计时秒数 |
| **红灯蜂鸣器** | 红灯亮时蜂鸣器鸣叫，绿灯亮时静音 |
| **按键调整时长** | RIGHT 键切换模式（运行→设置红灯→设置绿灯），UP/DOWN 键调整时长 (5-99s)，LEFT 键保存退出 |
| **显示布局** | 数码管 1：倒计时 / 数码管 2：红灯时长 / 数码管 3：绿灯时长 |
| **设置闪烁** | 设置模式下对应数码管数值以 1Hz 闪烁 |

**状态机结构**：
- `MODE_RUNNING` — 正常运行倒计时
- `MODE_SETTINGS_RED` — 设置红灯时长
- `MODE_SETTINGS_GREEN` — 设置绿灯时长
- 灯状态：`STATE_RED` / `STATE_GREEN` / `STATE_OFF`

---

### 4.2 模拟电梯系统

**核心设计模式**：有限状态机 (FSM)，非阻塞延时，定时器中断驱动，三层架构（驱动层/逻辑层/交互层）。

**功能需求**：

| 需求 | 描述 |
|---|---|
| **楼层选择** | 按键选择当前楼层和目标楼层（1-15 层） |
| **楼层显示** | 数码管显示当前楼层 |
| **运行指示** | 运行中红灯亮，到达楼层绿灯亮，蜂鸣器响 |
| **自动开门** | 到达楼层后等待 3 秒（模拟开门），超时自动进入下一状态 |
| **自动归位** | 任务完成后自动返回 1 楼 |

**8 状态状态机**：

| 状态 | 说明 |
|---|---|
| `STATE_IDLE` | 1 楼待机，等待用户 |
| `STATE_SELECT_CURRENT` | 选择起始楼层（数码管闪烁） |
| `STATE_SELECT_TARGET` | 选择目标楼层（数码管闪烁） |
| `STATE_MOVING_TO_USER` | 运行到接客楼层（红灯） |
| `STATE_DOOR_OPEN_USER` | 接客开门，等待 3 秒（黄灯） |
| `STATE_MOVING_TO_TARGET` | 送客运行（红灯） |
| `STATE_DOOR_OPEN_TARGET` | 送客开门，等待 3 秒（绿灯） |
| `STATE_RETURNING_HOME` | 自动返回 1 楼 |

---

### 4.3 直流电表数据采集

**注意**：由于实验无法提供 24V 供电的用电器，本模块只需完成与 HT7017 芯片通信、**采集裸数据**即可，无需计算真实物理值。

**功能需求**：

| 需求 | 描述 |
|---|---|
| **HT7017 初始化** | 配置 USART1（4800-9-Odd），初始化芯片，写 59H 寄存器 |
| **电压采样** | 调用 `HT7017_GetSignedSample()` 获取电压原始值 |
| **电流采样** | 调用 `HT7017_GetSignedSample()` 获取电流原始值 |
| **数码管显示** | 将电压、电流、功率、电能显示在数码管上 |
| **数据发送** | 通过 RS485 或 HC-05 蓝牙发送采样数据到上位机/手机 |

## 六、关键技术要点

1. **FSM 状态机** — 红绿灯和电梯的核心设计模式，所有状态切换在定时器中断中完成，主循环仅做 UI 刷新。
2. **逻辑与显示分离** — 主循环 `while(1)` **不包含业务逻辑**，只调用 UI 更新函数；核心逻辑全部在中断回调中。
3. **非阻塞设计** — 电梯移动、等待定时均通过 TIM6 中断驱动，不使用 `HAL_Delay()` 阻塞。
4. **USER CODE 区域** — 所有用户代码必须在 `USER CODE BEGIN/END` 注释区域内编写，否则 CubeMX 重新生成时会丢失。
5. **中断回调** — `HAL_TIM_PeriodElapsedCallback`（定时器）、`HAL_GPIO_EXTI_Callback`（按键）是核心逻辑入口。
